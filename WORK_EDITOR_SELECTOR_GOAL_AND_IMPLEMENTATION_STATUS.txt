==================================================
1) GOAL TO IMPLEMENT (TARGET BEHAVIOR)
==================================================

1.1 Product objective
- Build one consistent selection workflow where Work Editor uses the ACTUAL Tool Library and ACTUAL Jaws Library UIs.
- Do not maintain a second "look-alike" selector implementation.
- Keep existing library capability (search/filter/details/3D/edit actions) available, while adding assignment-oriented behavior for Work Editor.

1.2 UX principles
- Single source of truth: library pages and styles are reused, not duplicated.
- Assignment-first: users can build Work Editor assignments quickly without leaving library context.
- Predictable targeting: user always knows which HEAD/SPINDLE or SP slot is being edited.
- Reversible actions: clear visual feedback, safe cancel, no silent destructive behavior.

1.3 Scope in Work Editor entry points
- YLEISET tab:
  - "Tools & Jaws Selector" starts a chained flow:
    1) Tool selection stage
    2) Automatic transition to Jaw selection stage
    3) Return to Work Editor
- Tool IDs tab:
  - "Select Tools" opens tool selector stage only.
- Spindles/Jaws tab:
  - "Select Jaws" opens jaw selector stage only.
- Existing in-tab manual row edit actions remain available.

1.4 Tool selector target behavior (inside ACTUAL Tool Library window)
- Open behavior:
  - Opens real Tool Library module with selector session context.
  - Shows current target context (HEAD1/HEAD2 and SP1/SP2).
- Layout behavior:
  - Left side remains real tool catalog with existing search/filter/type/head behavior.
  - Right side changes from detail-only usage to assignment area behavior in selector mode.
- Assignment area requirements:
  - Supports drag from catalog -> assignment list.
  - Supports drag reorder within assignment list.
  - Supports drop insertion between existing rows (explicit insertion location indicator).
  - Supports replacing or appending behavior defined by drop position.
  - Supports multi-selection drag (when multiple tools selected).
- Duplicate policy:
  - Disallow duplicate same tool inside same (HEAD, spindle) bucket.
  - Allow same tool in different buckets.
- Row content requirements:
  - Mini-card row format consistent with Setup Card intent.
  - Shows order index, tool id/name, and optional meta (pot/comment/edit markers).
- Row edit requirements:
  - Double-click row opens row edit for:
    - comment
    - pot
    - override_id (T-code)
    - override_description
  - Edits persist in assignment payload returned to Work Editor.

1.5 Jaw selector target behavior (inside ACTUAL Jaws Library window)
- Open behavior:
  - Opens real Jaws Library module with selector session context.
- Assignment area requirements:
  - Two fixed targets: SP1 and SP2.
  - Drag jaw from catalog to SP1/SP2 target.
  - Re-drop on same slot replaces previous jaw selection.
- Stop-screw handling:
  - If selected jaw is spiked, stop-screw input is editable for that slot.
  - If jaw is not spiked, stop-screw input is hidden/disabled/ignored for that slot.
  - Existing stop-screw value is preserved unless explicitly changed.

1.6 Selector mode vs details mode
- Real library keeps details and 3D preview capability.
- In selector mode, right panel prioritizes assignment interaction.
- Details mode remains accessible (toggle or equivalent) without leaving selector session.
- Switching between selector/details modes must not lose current in-session assignment state.

1.7 Context and targeting rules
- Tool targeting context:
  - HEAD selection: HEAD1 or HEAD2.
  - Spindle selection: SP1(main) or SP2(sub).
  - Active context is clearly visible in selector UI.
- Jaw targeting context:
  - Active jaw slot context: SP1 or SP2.
- Default context resolution:
  - Uses launching tab/state (or current focused list) as default.
  - User can switch target context inside selector session.

1.8 Combined flow state machine (YLEISET button)
- Start -> ToolSelector(Open)
- ToolSelector:
  - Cancel => return to Work Editor, no chained jaw stage
  - Complete/Apply => JawSelector(Open)
- JawSelector:
  - Cancel => return to Work Editor; tool changes from previous stage remain (if already applied)
  - Complete/Apply => return to Work Editor
- Return state:
  - Work Editor refreshes lists and keeps current tab/state as sensible default.

1.9 Work Editor in-tab list visual target
- Tool rows in Work Editor must visually match mini-card language used in Setup Cards:
  - compact row height
  - consistent paddings and border radius
  - icon scale matching card typography
  - clean title/subtitle alignment (no overlap, clipping, or vertical drift)
  - selected state highlight consistent with catalog cards
- "No oversized/broken card layouts" is a hard acceptance requirement.

1.10 Data contract and persistence constraints
- Persisted work schema MUST stay unchanged:
  - head1_tool_assignments
  - head2_tool_assignments
  - main_jaw_id
  - sub_jaw_id
  - main_stop_screws
  - sub_stop_screws
- No DB migration.
- Preserve compatibility fields:
  - tool_uid when available
  - override fields
  - deleted-tool placeholder behavior

1.11 Save/cancel semantics
- Selector cancel:
  - must not apply new selection changes for that stage.
- Selector complete/apply:
  - applies stage results to Work Editor in-memory state.
- Work Editor final Save:
  - persists resulting state to DB.
- Work Editor Cancel:
  - discards in-memory changes from selector sessions opened in that dialog instance.

1.12 Error handling and resilience
- If external library is unavailable:
  - show clear warning and keep user in Work Editor.
- If callback payload is malformed:
  - show warning and do not corrupt current assignments.
- If selected items no longer exist in DB:
  - keep assignment id references and show missing/deleted visual marker as existing behavior expects.

1.13 Performance and responsiveness
- Opening selector from Work Editor should feel immediate (reuse running library instance when possible).
- Drag/drop and reordering operations should be smooth with no noticeable lag at normal list sizes.

1.14 Non-goals
- No database schema redesign.
- No removal of normal Tool Library/Jaws Library CRUD workflows.
- No replacement of existing Work Editor manual edit controls.

1.15 Definition of done (acceptance checklist)
- A) Tools selector from Work Editor opens real Tool Library.
- B) Jaws selector from Work Editor opens real Jaws Library.
- C) In selector mode, right-side assignment area supports:
  - drag add
  - drag insert between rows
  - drag reorder
- D) Tool row double-click editing works and persists in assignment payload.
- E) Duplicate rules are enforced per bucket exactly as specified.
- F) YLEISET combined flow chains Tools -> Jaws automatically on complete.
- G) SP1/SP2 jaw assignment and spiked-jaw stop-screw behavior work.
- H) Work Editor tool list cards visually match intended mini-card style and are not oversized/misaligned.
- I) Save/reload regression passes:
  - assignments
  - overrides
  - jaw selections
  - stop screws
  - print output compatibility.

A) Setup Manager -> External Library selector bridge
File: Setup Manager/ui/work_editor_dialog.py

Implemented:
- Removed runtime dependency on custom selector dialogs for active flow.
- Added local selector callback server per WorkEditorDialog instance (QLocalServer).
- Added request/session tracking with unique request_id per selector request.
- Added Tool Library IPC launch/send path:
  - Sends payload to tool_library_single_instance via QLocalSocket.
  - Payload includes selector mode metadata:
    - selector_mode (tools/jaws)
    - selector_callback_server
    - selector_request_id
    - selector_head (optional)
    - selector_spindle (optional)
- Added fallback launch and retry logic when Tool Library is not already running.
- Added callback payload handling:
  - command=selector_result validation
  - request_id correlation
  - kind-based routing (tools/jaws)
- Added assignment application logic:
  - Tools: appends selected tools to target (head, spindle) bucket
  - Duplicate prevention within same bucket by uid/id key
  - Preserves assignment schema fields (comment/pot/override fields initialized)
  - Same tool still allowed in different buckets
  - Jaws: applies selected jaw to requested spindle (main/sub)
- Added combined flow chaining:
  - YLEISET combined button now opens tools selector and auto-opens jaws selector on valid tool callback.
- Added selector bridge cleanup on close/destroy.

Also implemented:
- Local MiniAssignmentCard/tool-icon helpers moved into work_editor_dialog.py
  to keep row rendering functional without custom selector module dependency.
