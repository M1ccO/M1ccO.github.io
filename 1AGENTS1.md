\# Repository agent rules



\## General behavior

Prefer small, surgical edits over broad rewrites.

Preserve naming, module boundaries, and current UI patterns.

Do not invent architecture without checking existing files first.



\## Planning

When in plan mode:

\- identify likely files

\- describe responsibilities

\- propose the smallest safe refactor path

\- avoid coding unless asked



\## Building

When in build mode:

\- inspect relevant files before editing

\- prefer direct implementation over long explanation

\- keep diffs minimal

\- do not rename files or symbols unless necessary



\## Verification

After implementation:

\- summarize changed files

\- note likely breakpoints

\- suggest specific verification steps
